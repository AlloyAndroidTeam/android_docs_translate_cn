支持不同系统版本

Previous Next Get started

本章内容

  1. 指定最低及目标 API 级别
  2. 运行时检测系统版本
  3. 使用系统本身样式及主题风格（Styles and Themes）

参考阅读

  • Android API Levels
  • Android 支持库（Support Library）

Android新版本的通常有更高级的 API 可用，但是在此版本被广泛升级之前，你还是需要继续兼顾对旧版本的设备支持， 本章讲解如何在兼顾旧版本的同时又能享受新 API 所带来的便利。

平台版本 页面会定时更新Anrdoid每个版本在所有设备中的占比，数据源于 Google Play Store。一般而言， 在设定你的应用所支持的设备版本时，最好能覆盖到90%左右的设备。

提示： 如想跨多个版本使用新的特性和功能， 可在应用中包含 Android 支持库, 这样，你就可以在旧版本系统上使用较新的API了。

指定最低及目标 API 级别

AndroidManifest.xml 文件用于说明你的应用情况及其所支持的系统版本。其中， <uses-sdk 元素中的 minSdkVersion 与 targetSdkVersion 属性指定了你的应用所能支持的最低系统版本， 以及你的应用最高在哪个级别的API上进行了设计及测试。

例：

<manifest xmlns:android="http://schemas.android.com/apk/res/android" ... >
    <uses-sdk android:minSdkVersion="4" android:targetSdkVersion="15" />
    ...
</manifest>

与新的Android版本发布会带来新的样式和行为。如果想让你的应用能使用这些新特性，又想同兼顾所有版本上的样式， 可指定 targetSdkVersion 值为最新的 Android 版本

运行时检测系统版本

在 Build 这个常量类里，有Android每个系统版本号的值。你可以使用这些值来设定条件检测，以确保高版本API只在适合的系统中运行。

private void setUpActionBar() {
    // Make sure we're running on Honeycomb or higher to use ActionBar APIs
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.HONEYCOMB) {
        ActionBar actionBar = getActionBar();
        actionBar.setDisplayHomeAsUpEnabled(true);
    }
}

注意： Android在解析XML资源时，会忽略掉当前版本不支持的属性。所以， 使用新版本才支持的属性时不需担心在旧版本设备上会不会出问题。例如：设定 targetSdkVersion="11"时，你的应用在Android 3.0及以上的版本设备上默认会有 ActionBar，如此，如果你要把菜单放到 action bar 上， 需要在menu资源文件里设定 android:showAsAction="ifRoom"。这样写在不会出现版本兼容问题的，因为旧版本Android会直接忽略掉 showAsAction 属性 (也就是说, 你 不需要 把资源按版本目录 res/menu-v11/) 来区分资源。

使用系统本身样式及主题风格

Android提供了一些指定的主题风格。这些主题可以在 manifest 配置文件里使用。使用这些内建的主题， 你的应用可随着所在系统的主题风格变化而变化。（译注：android开发者基本上都自建一套主题吧， 没人愿意把主题风格交给默认这个看起来像工程机的系统来设定。）

弹出框：

<activity android:theme="@android:style/Theme.Dialog">

透明背景：

<activity android:theme="@android:style/Theme.Translucent">

使用在 /res/values/styles.xml 中自定义的风格：

<activity android:theme="@style/CustomTheme">

全应用范围应用主题风格可在 <application> 元素中设定 android:theme 

<application android:theme="@style/CustomTheme">

你可以在 样式与主题 一章中了解更多关于创建样式与主题的内容。
